Open Media Player — User actions

Reference for every interaction the player supports, grouped by where it happens. Every editable row (genre, artist, library, playlist, playlist track) carries a bold ⋯ kebab on its right edge that opens its action menu.

Layout in brief

  • Sources (left, resizable — drag its right edge): Libraries (checkboxes, top) and Playlists (fills the rest).
  • Browser cascade: Genres → Artists → Albums.
  • Tracklist + a status bar; the bar's far right shows the current list's track count and total time (e.g. 23 tracks · 1h 47m).
  • At any moment exactly one side is "active": either a playlist (highlighted in Sources) or the library cascade. Clicking a genre/artist/album snaps the active view back to the library; clicking a playlist row snaps it to that playlist.

Quality filters (gear menu → Filters…)

Hides low-quality archive.org results before they reach the album / track lists. Only the 3-minute track floor is on by default; the rest opt in. Catalog artists (specific /details/ collections) are not filtered unless you tick "also apply to catalog artists".

Where the data lives

A library is a self-contained container under libraries/<name>/ whose small index document (index.ttl) rdfs:seeAlso-links its sibling files:

  • agents.ttl — artists / groups / labels
  • genres.ttl — genres + the Music root concept
  • releases.ttl — an index that rdfs:seeAlso-links one file per album under releases/<slug> (each holding a mo:Release + its mo:Tracks)
  • playlists/<Slug> — one self-contained file per playlist

Each playlist file is self-contained: adding tracks writes the mo:Track + its parent mo:Release and the dcterms:hasPart edge into that one small playlist file — releases.ttl is not touched. A failed write rolls back so nothing half-lands; the status bar shows the error and nothing is silently lost.

Libraries whose URL is on another origin are listed but stay unchecked until you select them (then they load); only same-origin libraries auto-load at startup, so listing a remote library costs no network trip.

Session persistence

Saved to localStorage, restored next load:

  • Active source (Library or a playlist) and the playlist/library highlight.
  • Loaded track + playback position (Play resumes near where you left off).
  • Library track queue; sort column/direction; column widths; Sources column width.
  • Genre / Artist / Album selection.
  • Volume.
  • Registered libraries and which are enabled.
  • Quality-filter settings (its own key).
