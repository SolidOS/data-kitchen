Solid Data Kitchen - Using URI shortcuts

In the dataBrowser URI bar, the start of the URI and the end of the URI determine how the URI will be interpreted. Shortcuts use the localBase and remoteBase variables, see "How to customize Data Kitchen" for details on how to set them.

URI starts with

  • http - absolute remote or localhost web address
  • file - absolute local file system address
  • app - absolute in-browser address (e.g. IndexedDB)
  • / - (slash) - remote address relative to REMOTE_BASE
  • ./ - (dot slash) - local file system address relative to LOCAL_BASE
  • : - (colon) - address of an ontology (see below)
URI ends with
  • / - URI is a container (folder,directory)
  • anything else - URI is a resource (file,document)

Ontology shortcuts

  • If the URI is in the form :prefix:term, where prefix is an RDF prefix known to the Solid namespace, it will be expanded to the full URI. For example, :ldp:Container will bring you to http://www.w3.org/ns/ldp#Container. (note:in progress, not yet implemented)

Some examples

  • if REMOTE_BASE is set to "https://you.example.org/public/" the URI "/foo" will expand to "https://you.example.org/public/foo"
  • if LOCAL_BASE is set to "file:///home/me/data/" the URI "./foo" will expand to "file:///home/me/data/foo"
