Settings

Discovery-driven editor list. Every component on the page that declares static get shape() (or static get editor()) appears here as one collapsible panel. The list starts with every panel collapsed and re-runs discovery on each nav so widgets mounted later (e.g. by visiting another tab first) show up automatically.

Components that opt into inline editing (static get editor() { return { inline: true } }, e.g. sol-feed) are excluded — they edit themselves from their own widget.

How to use

Expand a panel — the editor mounts on first open. Save flows to the same TTL the component reads from, then the source widget is re-rendered automatically.

If the target TTL needs authentication, a login picker appears for the duration of the save. Once you're signed in the save retries and the panel returns to normal. (See the chrome help for the green-gear indicator.)

Menu uses a tree-edit view: drill into items to set per-item label, element tag, and attributes (e.g. source for sol-include items).
